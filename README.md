# Dentoi Website

Este projeto é uma **recriação completa de uma landing page profissional de clínica odontológica**, originalmente inspirada em um template do Framer, porém **desenvolvida manualmente utilizando HTML, CSS e JavaScript**, com foco em organização de código, reutilização de componentes e boas práticas de front-end.

O objetivo do projeto é demonstrar habilidades em **estruturação de layout, componentização, responsividade e performance web**.

---

---

## 📂 Estrutura de Pastas

O projeto foi organizado para facilitar manutenção e reutilização de estilos:


### 🔹 Organização do CSS

- **components/** → Estilos reutilizáveis (botões, formulários, header, navbar)  
- **default/** → Estilos globais e utilitários (reset, layout base)
- **styles-pags/** → Estilos específicos de páginas  
- **styles.css** → Arquivo principal que importa todos os módulos  

Essa estrutura segue um padrão modular inspirado em **Atomic Design + CSS Architecture (SMACSS)**.

## 🎯 Performance & Qualidade de Código

Durante o desenvolvimento do projeto, foi utilizado o **Google Lighthouse como ferramenta de apoio contínuo**, com o objetivo de garantir **qualidade de código, boa performance e aderência às boas práticas de front-end**.

As auditorias foram realizadas em **todas as páginas**:

- Home
- Services
- Blog
- Contact
- Páginas institucionais (Policies)

---

### 🔍 Métricas Avaliadas

As análises contemplaram os quatro pilares do Lighthouse:

- **Performance**
- **Accessibility**
- **Best Practices**
- **SEO**


Os resultados apresentaram **métricas consistentes e positivas em todas as seções**, tanto em **Desktop quanto em Mobile**, refletindo um cuidado real com experiência do usuário e qualidade técnica.

---

### ⚙️ Decisões Técnicas Baseadas em Métricas

Os relatórios do Lighthouse foram utilizados para orientar decisões durante o desenvolvimento, como:

- Organização semântica do HTML
- Criação de **CSS modular e reutilizável**
- Separação clara entre estilos globais, componentes e páginas
- Redução de CSS redundante
- Aplicação de **lazy loading** em imagens
- Otimização de imagens e assets
- Evitar recursos que bloqueiam a renderização
- Estrutura correta de headings para acessibilidade e SEO
- Garantia de contraste e legibilidade

---

### 📱 Experiência Mobile

As métricas foram avaliadas com foco em **dispositivos móveis**, priorizando:

- Layout responsivo
- Performance em conexões mais lentas
- Boa experiência de navegação em telas menores
